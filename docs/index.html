Typeless

按住說話，AI 幫你打字
macOS Python Groq Whisper 中英混合

是什麼

macOS 上的語音輸入工具。按住 Option + Space 說話，放開後 AI 潤飾過的文字直接貼到目前的視窗。

取代 Apple 內建語音輸入的痛點：中英不用切換、AI 自動加標點修錯字、根據當前 App 調整語氣（寫 email 正式、Slack 輕鬆）。

核心功能

🎤
Push-to-talk

按住 Option+Space，像對講機一樣直覺。

🌏
中英混合

「這個 API 的 endpoint 是什麼」直接打出來，不用切輸入法。

🎨
App 自動語氣

偵測當前 App，調整潤飾風格（Mail 正式 / Slack 輕鬆 / VS Code 技術）。

🗣️
語音指令

說「重新說」執行 Cmd+Z；「全選」、「換行」、「複製」皆可。

⚡
低延遲

Groq Whisper + LLaMA 3.3，放開到貼上約 1.5 秒。

🚀
開機自啟

LaunchAgent 在背景跑，不用每次手動開。

🐼
熊貓浮動視窗

三種風格可選（capsule 膠囊 / card 卡片 / bubble 手繪熊貓臉），螢幕上方顯示 ● Recording → Analyzing → 結果文字。

🔇
幻覺過濾

音量 RMS 偵測 + 黑名單，避開 Whisper 把靜音猜成「謝謝觀看」之類的訓練資料噪音。

📤
送出指令

「送出」「傳送」「send」直接按 Enter，不用切回鍵盤。

    ▼

    ▼

    ▼

    ▼

檔案角色
main.py主程式 / menu bar

output.py剪貼板貼上 + 鍵盤指令

安裝

三步驟：

# 1. 複製程式
git clone https://github.com/chingyaosu/Typeless.git ~/Desktop/CLAUDE/Typeless

# 2. 安裝相依
pip install -r ~/Desktop/CLAUDE/Typeless/requirements.txt

# 3. 設定 API key
echo "GROQ_API_KEY=你的_key" > ~/.typeless_env

剩下的（建立 .app bundle、LaunchAgent、權限）請見 完整 INSTALL.md。

遇到的坑（給後人）

1. macOS Gatekeeper 擋 ad-hoc 簽名 .app

解法：用 LaunchAgent 跑，不靠 .app 雙擊啟動。

2. python3 不能加進「輔助使用」

新版 macOS 只能加 .app bundle。解法：複製 /opt/anaconda3/python.app 改 Info.plist 變成 Typeless.app，bundle ID 設 com.typeless.app。

3. 一直辨識成 "THANK YOU"

Whisper 收到靜音時的幻覺。原因：python.app 的 Info.plist 沒有 NSMicrophoneUsageDescription，macOS 給了「沉默音訊」。解法：在自訂 Info.plist 加上這個 key。

4. Tink 開始音延遲到放開後才響

原因：sounddevice 開錄音搶走了 audio session。解法：app 啟動時就把 InputStream 開好持續執行，不要每次按鍵才開。

5. 按一陣子後 Option+Space 失效（變成插入空白）

CGEventTap 被 macOS 因為 callback 太慢而暫時停用。解法：在 callback 裡偵測 kCGEventTapDisabledByTimeout 並自動重啟。

6. Whisper 對短音訊亂猜成韓文／日文／中國人名

「送出」兩個字會被 Whisper 聽成 "Sączu." 或「澤東 曹 公豐」。解法：API 加 language="zh" 鎖定中文（仍能保留英文字）+ 中英雙語 prompt 提示常用指令。

7. 安靜時 Whisper 幻覺出 YouTube 字幕

常見幻覺：「謝謝觀看,下次見!」、「全文字幕由 Amara.org 社群提供」。原因：Whisper 訓練資料含大量 YouTube 字幕。解法：錄音時計算 RMS 音量，< 200 直接丟棄不送 API。

8. 浮動視窗從背景 thread 更新會 crash

AppKit 強制 UI 必須在主執行緒。解法：用 NSObject 子類 + performSelectorOnMainThread_ 把 closure 派送到主執行緒。
